  └── docker compose up
      ├── db        (postgres:16)           ← healthcheck gate
      ├── backend   (FastAPI / uvicorn)     ← depends on db
      ├── frontend  (nginx static)          ← depends on backend
      └── worker    (Python job)            ← depends on backend + db
Key differences vs. real app
  • No Copernicus / CMEMS credentials needed
  • No large NetCDF or FES2022 tide model files
  • No GEBCO bathymetry data volume mount
  • No real JWT secret (uses dummy token)
  • DB schema is empty — no migrations needed
  • Worker just prints steps and exits (no real processing)
  • All API responses are mocked in-memory
